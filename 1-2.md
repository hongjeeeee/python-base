# 알고리즘 사례
다음에서 해결하고자 하는 어떤 문제에 대한 알고리즘의 적용 예를 보여주고 있다. 여기서 문제의 정의는 **코끼리를 냉장고에 넣은 것**이다. 
- 이 문제에 대한 알고리즘을 다음과 같이 정의한다.
  - 냉장고의 문을 연다. 
  - 코끼리를 냉장고에 넣는다. 
  - 냉장고의 문을 닫는다.

이 문제를 위해서 실제로 실행되는 것이 아닌 의사 코드(pseudo code)로 위의 알고리즘을 코딩해 보면 다음과 같으며 open_door, put_elephant, close_door의 세 가지의 작업을 사용하여 다음과 같이 프로그램에서 구현하였다. 
```python
open_door()
put_elephant()
close_door()
```
이 세 가지 작업들은 정확하게 순서대로 진행되어야 하는데 그 이유는 순서가 바뀌면 프로그램은 원하는 결과를 내어 주지 못하기 때문이다. 
이번에는 같은 냉장고에 하마(hippo)를 넣는 알고리즘을 알아보기로 하자. 문제의 정의는 **하마를 냉장고에 넣는 것**이다. 
- 이 문제에 대한 알고리즘을 다음과 같이 네 단계로 정의한다. 
  - 냉장고의 문을 연다
  - 코끼리를 꺼낸다.
  - 하마를 냉장고에 넣는다.
  - 냉장고 문을 닫는다.

위의 알고리즘을 구현한 의사 코드는 다음과 같다. 
```python
open_door()
take_out_elephant()
put_hippo()
close_door()
```
만일 알고리즘에서 실행 순서가 바뀌거나 빠지면 원하는 결과를 얻을 수 없게 된다. 그러므로 프로그래밍에 입문하는 독자들은 컴퓨터의 문제 해결 방식인 알고리즘에 관심을 두고 차분하게 따라갈 수 있도록 많은 연습을 해야 한다.
